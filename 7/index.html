1) Pertemuan 4 ini membahas tentang Class Diagram :
-Class Diagram menggambarkan hubungan antar class atau tabel (struktur tabel )
-Class diagram terdiri dari :
- Nama class atau nama tabel diusahakan jangan pakai spasi
- Attribut diisi dengan field yang menjadi key, nama field,tipe data,lebar atau panjang data
- Method diisi dengan operasi atau hubungan dengan basis data tersebut misal: select/baca, insert/tambah/simpan dst
-Class Diagram itu bisa dibilang referensi lain dari Entity Relationship Diagram (ERD) dan Logical Record Structure (LRS).
2) Stereotype = menentukan/menandakan attribute yang menjadi pk
3) Multiplicipy = menentukan tingkat hubungan/kardinalitas antar class melalui association
